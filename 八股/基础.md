# Transformer Attention

## 1. Attention 要解决什么问题？

Attention 的核心目标是：

> 对于序列中的每个 token，动态地判断它应该关注序列中的哪些其他 token。

例如句子：

> The animal didn't cross the street because it was tired.

模型在理解 `it` 时，需要判断 `it` 更可能指代 `animal`，而不是 `street`。  
Attention 就是用来建模这种 **token 与 token 之间依赖关系** 的机制。

---

## 2. Q、K、V 分别是什么？

Transformer Attention 中最核心的是三个矩阵：

| 名称 | 全称 | 含义 |
|---|---|---|
| Q | Query | 当前 token 想找什么信息 |
| K | Key | 每个 token 提供什么索引信息 |
| V | Value | 每个 token 实际携带的内容信息 |

可以用检索系统类比：

- Query：你输入的搜索请求
- Key：数据库中文档的关键词
- Value：真正返回给你的文档内容

在 Self-Attention 中，Q、K、V 都来自同一个输入序列，只是通过不同的线性变换得到。

假设输入为：

\[
X \in \mathbb{R}^{n \times d_{model}}
\]

其中：

- \(n\)：序列长度
- \(d_{model}\)：每个 token 的隐藏维度

则：

\[
Q = XW_Q
\]

\[
K = XW_K
\]

\[
V = XW_V
\]

其中：

\[
W_Q, W_K, W_V \in \mathbb{R}^{d_{model} \times d_k}
\]

---

## 3. Scaled Dot-Product Attention

标准 Attention 公式是：

\[
Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V
\]

可以分成四步理解。

---

### 3.1 计算相关性分数

\[
QK^T
\]

如果：

\[
Q \in \mathbb{R}^{n \times d_k}
\]

\[
K \in \mathbb{R}^{n \times d_k}
\]

那么：

\[
QK^T \in \mathbb{R}^{n \times n}
\]

这个矩阵表示：

> 每个 token 对其他所有 token 的关注程度。

第 \(i\) 行第 \(j\) 列表示：

> 第 \(i\) 个 token 对第 \(j\) 个 token 的注意力打分。

---

### 3.2 为什么要除以 \(\sqrt{d_k}\)？

公式中有一项：

\[
\frac{QK^T}{\sqrt{d_k}}
\]

原因是：当 \(d_k\) 较大时，点积 \(QK^T\) 的数值可能变得很大。

如果输入 softmax 的值过大，softmax 会变得非常尖锐，例如：

\[
softmax([1,2,20]) \approx [0,0,1]
\]

这样会导致梯度很小，不利于训练。

所以除以 \(\sqrt{d_k}\) 是为了稳定数值范围，使训练更稳定。

---

### 3.3 Softmax 得到注意力权重

\[
A = softmax(\frac{QK^T}{\sqrt{d_k}})
\]

其中：

\[
A \in \mathbb{R}^{n \times n}
\]

每一行的和为 1。

也就是说，对于每个 token，它会对所有 token 分配一组权重。

例如：

```text
当前 token = "it"

attention weights:
animal: 0.65
street: 0.10
because: 0.05
tired: 0.20
```

这表示模型认为 `it` 更应该关注 `animal`。

---

### 3.4 用注意力权重加权求和 Value

最后一步：

\[
A V
\]

如果：

\[
A \in \mathbb{R}^{n \times n}
\]

\[
V \in \mathbb{R}^{n \times d_k}
\]

那么：

\[
AV \in \mathbb{R}^{n \times d_k}
\]

每个 token 的新表示，就是所有 token 的 Value 按注意力权重加权求和得到的。

---

## 4. Self-Attention 是什么？

Self-Attention 指的是：

> Q、K、V 都来自同一个序列。

例如输入句子：

```text
I love machine learning
```

每个 token 都会和其他 token 计算关系：

```text
I        -> I, love, machine, learning
love     -> I, love, machine, learning
machine  -> I, love, machine, learning
learning -> I, love, machine, learning
```

所以 Self-Attention 可以直接建模长距离依赖。

相比 RNN，Transformer 不需要一步一步顺序处理 token，而是可以并行计算所有 token 之间的关系。

---

## 5. Attention 的矩阵维度推导

假设：

* batch size = (B)
* sequence length = (n)
* hidden size = (d_{model})
* attention head 维度 = (d_k)

输入：

\[
X \in \mathbb{R}^{B \times n \times d_{model}}
\]

线性映射后：

\[
Q,K,V \in \mathbb{R}^{B \times n \times d_k}
\]

计算 attention score：

\[
QK^T \in \mathbb{R}^{B \times n \times n}
\]

softmax 后：

\[
A \in \mathbb{R}^{B \times n \times n}
\]

乘以 (V)：

\[
AV \in \mathbb{R}^{B \times n \times d_k}
\]

输出维度与 (Q)、(K)、(V) 的最后一维一致。

---

## 6. Multi-Head Attention

单个 attention head 只能在一个表示子空间中学习关系。

Multi-Head Attention 的思想是：

> 用多个 attention head 从不同角度建模 token 之间的关系。

公式为：

\[
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
\]

然后拼接多个 head：

\[
MultiHead(Q,K,V)=Concat(head_1,...,head_h)W_O
\] 

其中：

* (h)：head 数量
* 每个 head 的维度通常为：

\[
d_k = \frac{d_{model}}{h}
\]

例如：

```text
d_model = 512
num_heads = 8
每个 head 的维度 = 64
```

每个 head 可能关注不同类型的信息：

| Head   | 可能学习到的关系 |
| ------ | -------- |
| Head 1 | 主谓关系     |
| Head 2 | 指代关系     |
| Head 3 | 局部短语关系   |
| Head 4 | 长距离依赖    |
| Head 5 | 位置关系     |
| Head 6 | 语义相似性    |
| Head 7 | 句法结构     |
| Head 8 | 特定任务相关模式 |

注意：这只是直观解释，实际每个 head 学到的内容不一定完全可解释。

---

## 7. Masked Attention

Attention 中经常会用到 mask。

mask 的作用是：

> 禁止模型关注某些位置。

常见 mask 有两种。

---

### 7.1 Padding Mask

输入序列通常会 padding 到同样长度。

例如：

```text
句子1: I love NLP
句子2: I love deep learning very much
```

为了组成 batch，句子1可能被 padding：

```text
I love NLP [PAD] [PAD] [PAD]
```

Padding Mask 用来防止模型关注 `[PAD]`。

---

### 7.2 Causal Mask / Look-ahead Mask

在 GPT 这种自回归语言模型中，预测当前位置时不能看到未来 token。

例如：

```text
I love machine learning
```

当模型预测 `machine` 时，只能看：

```text
I love
```

不能看：

```text
learning
```

因此需要 causal mask。

Attention score 矩阵中，未来位置会被 mask 掉：

```text
可见位置矩阵：

1 0 0 0
1 1 0 0
1 1 1 0
1 1 1 1
```

第 (i) 个 token 只能关注自己以及之前的 token。

---

## 8. Encoder Attention 和 Decoder Attention

Transformer 原始结构包括 Encoder 和 Decoder。

---

### 8.1 Encoder Self-Attention

Encoder 中使用的是双向 Self-Attention。

每个 token 可以看到整个输入序列。

适合：

* 文本分类
* 序列标注
* 语义理解
* BERT 类模型

例如：

```text
The cat sat on the mat
```

每个 token 都可以关注左右两边的所有 token。

---

### 8.2 Decoder Masked Self-Attention

Decoder 中使用的是 Masked Self-Attention。

每个 token 只能看到当前位置之前的信息。

适合：

* 文本生成
* 语言模型
* GPT 类模型

例如生成：

```text
I love
```

下一步预测：

```text
machine
```

模型不能提前看到 `machine` 后面的内容。

---

### 8.3 Cross-Attention

在 Encoder-Decoder 结构中，Decoder 还会使用 Cross-Attention。

Cross-Attention 中：

* Q 来自 Decoder
* K、V 来自 Encoder

公式仍然是：

\[
Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V
\]

但来源不同：

```text
Q: decoder hidden states
K: encoder hidden states
V: encoder hidden states
```

Cross-Attention 用于让 Decoder 在生成时关注输入序列。

常见于：

* 机器翻译
* 文本摘要
* 图像描述生成
* 多模态模型中的跨模态对齐

---

## 9. Attention 为什么需要位置编码？

Self-Attention 本身不包含顺序信息。

例如下面两个句子：

```text
I love you
You love I
```

如果没有位置编码，Attention 只看到 token 集合，很难区分顺序。

所以 Transformer 需要加入位置编码。

常见位置编码方式包括：

| 方法     | 说明               |
| ------ | ---------------- |
| 绝对位置编码 | 给每个位置一个固定或可学习向量  |
| 相对位置编码 | 建模 token 之间的相对距离 |
| RoPE   | 旋转位置编码，常用于大语言模型  |
| ALiBi  | 通过注意力偏置表达距离衰减    |

原始 Transformer 使用的是正弦余弦位置编码：

\[
PE(pos,2i)=sin(\frac{pos}{10000^{2i/d_{model}}})
\]

\[
PE(pos,2i+1)=cos(\frac{pos}{10000^{2i/d_{model}}})
\]

其中：

* (pos)：位置
* (i)：维度索引

---

## 10. Attention 的计算复杂度

Self-Attention 的主要开销来自：

\[
QK^T
\]

如果序列长度为 (n)，隐藏维度为 (d)，则复杂度大致为：
\[
O(n^2 d)
\]

空间复杂度主要来自 attention matrix：

\[
A \in \mathbb{R}^{n \times n}
\]

所以空间复杂度是：

\[
O(n^2)
\]

这也是 Transformer 处理长序列时成本很高的原因。

例如：

```text
n = 1024
attention matrix 大小 = 1024 × 1024

n = 8192
attention matrix 大小 = 8192 × 8192
```

序列长度增加 8 倍，attention matrix 大小增加 64 倍。

---

## 11. 常见 Attention 改进方向

由于标准 Attention 是 (O(n^2))，长序列场景下会很贵，因此有很多改进方法。

### 11.1 Sparse Attention

只让每个 token 关注部分 token。

例如：

* 局部窗口 attention
* block attention
* sliding window attention
* global token attention

优点：

```text
降低计算量
```

缺点：

```text
可能损失全局依赖建模能力
```

---

### 11.2 Linear Attention

希望把复杂度从：

\[
O(n^2)
\]

降到：

\[
O(n)
\]

基本思想是避免显式计算完整的 (n \times n) attention matrix。

---

### 11.3 FlashAttention

FlashAttention 不是改变 attention 数学公式，而是优化计算方式。

它的核心目标是：

> 减少 GPU HBM 显存读写，提高 attention 计算效率。

标准 Attention 会显式存储较大的 attention matrix，而 FlashAttention 通过分块计算和 IO 优化减少显存开销。

优点：

* 更快
* 更省显存
* 结果与标准 attention 数学上等价或近似等价，取决于具体实现

---

### 11.4 Multi-Query Attention, MQA
【注意力革命：GQA/MLA/DSA如何让模型更高效 - Jia-Bin Huang】 https://www.bilibili.com/video/BV1kudFBkE3Y/?share_source=copy_web&vd_source=c85963c68edeae09bd8fd784312c5ebc

标准 Multi-Head Attention 中，每个 head 都有自己的 Q、K、V。

MQA 的做法是：

* 多个 head 使用不同的 Q
* 但共享 K 和 V

这样可以减少 KV Cache 的大小，提升推理速度。

---

### 11.5 Grouped-Query Attention, GQA

GQA 介于 MHA 和 MQA 之间。

它不是所有 head 共享一组 K、V，而是多个 query head 分成若干组，每组共享一组 K、V。

对比：

| 方法  | Q          | K/V        | 特点            |
| --- | ---------- | ---------- | ------------- |
| MHA | 每个 head 独立 | 每个 head 独立 | 表达能力强，但推理开销大  |
| MQA | 多个 Q head  | 共享一组 K/V   | 推理快，但可能损失表达能力 |
| GQA | 多个 Q head  | 分组共享 K/V   | 折中方案          |

## 12. KV Cache 是什么？

在自回归生成中，模型一次生成一个 token。

例如：

```text
I
I love
I love machine
I love machine learning
```

如果每次都重新计算前面所有 token 的 K、V，会非常浪费。

KV Cache 的做法是：

> 把历史 token 的 K、V 缓存起来，下一步生成时直接复用。

这样推理时只需要计算新 token 的 Q、K、V，并与缓存中的 K、V 做 attention。

优点：

```text
显著加速自回归生成
```

缺点：

```text
长上下文时 KV Cache 占用大量显存
```

KV Cache 的大小与以下因素有关：

* batch size
* sequence length
* layer 数量
* head 数量
* head dimension
* 数据精度，例如 FP16、BF16、INT8

---

## 13. Attention 和全连接层的区别

Attention 不是普通的全连接层。

全连接层的权重是固定参数：

```text
输出 = 输入 × 参数矩阵
```

而 Attention 的权重是根据输入动态计算出来的：

```text
attention weight = softmax(QK^T / sqrt(d_k))
```

也就是说：

> Attention 的连接强度会根据不同输入样本动态变化。

这使得模型可以灵活建模上下文关系。

---

## 14. Attention 和 CNN / RNN 的对比

| 模型        | 特点   | 优点            | 缺点           |
| --------- | ---- | ------------- | ------------ |
| CNN       | 局部卷积 | 并行高效，适合局部模式   | 长距离依赖需要堆很多层  |
| RNN       | 顺序递归 | 天然建模序列        | 难并行，长距离依赖困难  |
| Attention | 全局交互 | 可并行，直接建模长距离关系 | 复杂度 (O(n^2)) |

Attention 的优势在于：

```text
任意两个 token 可以一层内直接交互。
```

## 15. Transformer Block 中 Attention 的位置

一个典型 Transformer Block 包括：

```text
Input
  ↓
LayerNorm
  ↓
Multi-Head Self-Attention
  ↓
Residual Connection
  ↓
LayerNorm
  ↓
Feed Forward Network
  ↓
Residual Connection
  ↓
Output
```

注意：

* Attention 负责 token 之间的信息交互
* FFN 负责对每个 token 的表示做非线性变换
* Residual Connection 缓解深层网络训练困难
* LayerNorm 稳定训练

现代大模型常用 Pre-LN 结构：

```text
x = x + Attention(LN(x))
x = x + FFN(LN(x))
```

## 16. PyTorch 风格伪代码

```python
import torch
import torch.nn.functional as F

def attention(Q, K, V, mask=None):
    """
    Q: [B, heads, n, d_k]
    K: [B, heads, n, d_k]
    V: [B, heads, n, d_k]
    mask: [B, 1, n, n] or [1, 1, n, n]
    """

    d_k = Q.size(-1)

    # 1. compute attention scores
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    # scores: [B, heads, n, n]

    # 2. apply mask
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    # 3. softmax
    attn_weights = F.softmax(scores, dim=-1)

    # 4. weighted sum
    output = torch.matmul(attn_weights, V)
    # output: [B, heads, n, d_k]

    return output, attn_weights
```

## 17. 常见面试 / 考试问题

### Q1：为什么 Attention 要除以 (\sqrt{d_k})？

因为 (QK^T) 的点积值会随着维度 (d_k) 增大而变大。
如果不缩放，softmax 容易饱和，导致梯度变小，训练不稳定。

---

### Q2：Self-Attention 和 Cross-Attention 的区别是什么？

Self-Attention：

```text
Q, K, V 来自同一个序列
```

Cross-Attention：

```text
Q 来自目标序列
K, V 来自源序列
```

例如机器翻译中：

```text
Encoder 输入英文句子
Decoder 生成中文句子
```

Decoder 的 Cross-Attention 会让中文生成过程关注英文输入。
此时 Self-Attention 的 Q、K、V 来源取决于当前是在 Encoder 还是 Decoder 中。

---

### Q3：Multi-Head Attention 为什么有用？

因为不同 head 可以在不同子空间中学习不同类型的关系。

例如：

* 一个 head 关注局部短语
* 一个 head 关注长距离依赖
* 一个 head 关注指代关系
* 一个 head 关注句法结构

多头机制提升了表达能力。

---

### Q4：为什么 Transformer 需要位置编码？

因为 Self-Attention 本身对顺序不敏感。

如果没有位置编码，模型难以区分：

```text
dog bites man
man bites dog
```

所以必须注入位置信息。

---

### Q5：Attention 的复杂度是多少？

标准 Self-Attention 的时间复杂度是：

\[
O(n^2d)
\]

空间复杂度是：

\[
O(n^2)
\]

其中：

* (n)：序列长度
* (d)：隐藏维度

主要瓶颈是 (n \times n) 的 attention matrix。

---

### Q6：Causal Mask 的作用是什么？

Causal Mask 用于自回归生成，防止模型看到未来 token。

例如预测第 (t) 个 token 时，只允许模型关注：

```text
1, 2, ..., t-1
```

不能关注：

```text
t+1, t+2, ...
```

---

### Q7：KV Cache 为什么能加速推理？

因为自回归生成时，历史 token 的 K、V 不会变化，可以缓存起来。
下一步生成时只需要计算新 token 的 Q、K、V，然后与缓存的 K、V 做 attention。

# 损失函数与优化器

## 1. 模型训练到底在做什么？

深度学习训练的核心过程可以概括为：

```text
输入数据 → 模型预测 → 计算损失 → 反向传播 → 优化器更新参数
```

也就是：

1. 模型根据输入 (x) 得到预测结果 (\hat{y})
2. 用损失函数比较 (\hat{y}) 和真实标签 (y) 的差距
3. 通过反向传播计算每个参数对损失的影响
4. 优化器根据梯度更新模型参数
5. 重复很多轮，使损失逐渐下降

---

## 2. 损失函数是什么？

损失函数的作用是：

> 衡量模型预测结果和真实答案之间的差距。

记作：

\[
L(y, \hat{y})
\]

其中：

* (y)：真实标签
* (\hat{y})：模型预测
* (L)：损失值

损失越小，说明模型预测越接近真实结果。

---

## 3. 损失函数和评价指标的区别

很多人会混淆 **loss** 和 **metric**。

| 对比项      | 损失函数 Loss         | 评价指标 Metric            |
| -------- | ----------------- | ---------------------- |
| 作用       | 用来训练模型            | 用来评价模型效果               |
| 是否参与反向传播 | 是                 | 一般不是                   |
| 是否需要可导   | 通常需要              | 不一定                    |
| 例子       | Cross Entropy、MSE | Accuracy、F1、BLEU、ROUGE |

例如分类任务中：

* 训练时通常用 **交叉熵损失**
* 评估时通常看 **准确率 Accuracy**

准确率本身不可导，所以不能直接作为常规神经网络训练的损失函数。

## 4. 常见损失函数

### 4.1 均方误差 MSE Loss

常用于回归任务。

例如预测房价、温度、坐标、连续值。

公式：

\[
MSE = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2
\]

含义：

> 预测值和真实值差距越大，惩罚越大。

例如：

```text
真实值 y = 10
预测值 y_hat = 8
误差 = 2
平方误差 = 4
```

MSE 的特点：

* 对大误差非常敏感
* 适合连续值预测
* 容易被异常值影响

---

### 4.2 平均绝对误差 MAE Loss

公式：
\[
MAE = \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|
\]

特点：

* 相比 MSE，对异常值不那么敏感
* 梯度相对不如 MSE 平滑
* 也常用于回归任务

对比：

| Loss | 对大误差的惩罚 | 是否容易受异常值影响 |
| ---- | ------- | ---------- |
| MSE  | 很强      | 是          |
| MAE  | 线性惩罚    | 较弱         |

---

### 4.3 交叉熵损失 Cross Entropy Loss

这是分类任务最常用的损失函数。

例如：

```text
输入：一张猫的图片
真实标签：猫
模型预测：
猫：0.8
狗：0.1
鸟：0.1
```

如果真实类别是“猫”，那么模型给“猫”的概率越高，loss 越低。

交叉熵公式：

\[
CE = -\sum_i y_i \log(\hat{y}_i)
\]

如果是真实类别 (c)，可以简化为：

\[
CE = -\log(\hat{y}_c)
\]

其中：

* (\hat{y}_c)：模型分给正确类别的概率

例如：

```text
正确类别概率 = 0.9
loss = -log(0.9) ≈ 0.105

正确类别概率 = 0.1
loss = -log(0.1) ≈ 2.303
```

所以：

> 模型越不相信正确答案，交叉熵损失越大。

---

### 4.4 Binary Cross Entropy, BCE

BCE 用于二分类或多标签分类。

公式：

\[
BCE = -[y\log(\hat{y}) + (1-y)\log(1-\hat{y})]
\]

适合：

```text
二分类：
是否垃圾邮件
是否患病
是否点击广告
```

也适合多标签分类：

```text
一张图片可能同时包含：
猫、沙发、窗户
```

注意：

* 多分类：通常用 `CrossEntropyLoss`
* 多标签：通常用 `BCEWithLogitsLoss`

---

### 4.5 Negative Log Likelihood Loss, NLL Loss

NLL Loss 本质上也是分类损失。

如果模型输出的是 log probability，那么可以使用 NLL Loss。

在 PyTorch 中：

```python
CrossEntropyLoss = LogSoftmax + NLLLoss
```

也就是说：

```python
nn.CrossEntropyLoss()
```

内部已经包含了 `softmax` 相关操作。

所以使用 PyTorch 的 `CrossEntropyLoss` 时，模型最后一层通常不要手动加 softmax。

---

### 4.6 Contrastive Loss / InfoNCE Loss

常用于对比学习、检索、多模态对齐。

目标是：

> 让正样本更接近，让负样本更远离。

例如图文对齐：

```text
图片：一只猫在沙发上
正文本：a cat on the sofa
负文本：a dog in the park
```

模型应该让图片表示更接近正确文本，而远离错误文本。

CLIP、SimCLR、很多多模态模型都会用类似思想。

---

### 4.7 语言模型中的损失函数

GPT 这类语言模型通常用 **next-token prediction loss**。

任务是：

```text
输入：I love machine
目标：learning
```

模型需要预测下一个 token。

本质上还是分类任务：

> 在整个词表中预测下一个 token 是谁。

如果词表大小是 50,000，那么模型每一步都在做一个 50,000 类分类。

因此语言模型常用：

```text
Cross Entropy Loss
```

例如句子：

```text
I love machine learning
```

训练样本可以看成：

```text
输入: I
预测: love

输入: I love
预测: machine

输入: I love machine
预测: learning
```

最终 loss 是所有位置的交叉熵平均。

---

## 5. 优化器是什么？

优化器的作用是：

> 根据梯度更新模型参数，使损失函数下降。

假设模型参数是 (\theta)，损失函数是 (L(\theta))。

优化目标是：

\[
\min_\theta L(\theta)
\]

最基本的参数更新形式是：

\[
\theta = \theta - \eta \nabla_\theta L
\]

其中：

* (\theta)：模型参数
* (\eta)：学习率 learning rate
* (\nabla_\theta L)：损失函数对参数的梯度

直观理解：

```text
梯度告诉你 loss 上升最快的方向
负梯度方向就是 loss 下降最快的方向
```

所以优化器沿着负梯度方向更新参数。

---

## 6. 常见优化器

### 6.1 SGD

SGD，全称 Stochastic Gradient Descent，随机梯度下降。

更新公式：

\[
\theta_{t+1} = \theta_t - \eta g_t
\]

其中：

* (g_t)：当前 batch 上的梯度
* (\eta)：学习率

优点：

* 简单
* 泛化能力较好
* 理论基础清晰

缺点：

* 收敛可能较慢
* 对学习率比较敏感
* 容易震荡

---

### 6.2 Momentum

Momentum 在 SGD 基础上加入“动量”。

直观理解：

> 不只看当前梯度，还参考之前更新的方向。

类似小球下山：

```text
如果之前一直往某个方向走，
那么下一步也倾向于继续往这个方向走。
```

更新形式：

\[
v_t = \beta v_{t-1} + g_t
\]

\[
\theta_{t+1} = \theta_t - \eta v_t
\]

优点：

* 减少震荡
* 加速收敛
* 对狭长损失曲面更友好

---

### 6.3 AdaGrad

AdaGrad 会根据历史梯度自动调整每个参数的学习率。

对于经常更新的参数，学习率变小；
对于不常更新的参数，学习率相对较大。

适合稀疏特征场景。

缺点是：

> 学习率会不断累积变小，后期可能几乎不更新。

---

### 6.4 RMSProp

RMSProp 改进了 AdaGrad 学习率不断变小的问题。

它使用梯度平方的滑动平均，而不是一直累加所有历史梯度。

适合非平稳目标，比 AdaGrad 更稳定。

---

### 6.5 Adam

Adam 是目前深度学习中最常用的优化器之一。

它结合了：

```text
Momentum + RMSProp
```

Adam 同时维护：

1. 一阶矩估计：梯度的滑动平均
2. 二阶矩估计：梯度平方的滑动平均

直观理解：

> Adam 不仅考虑当前梯度方向，还会自动调整每个参数的学习率。

优点：

* 收敛快
* 对学习率不那么敏感
* 适合 Transformer、CNN、RNN 等大多数神经网络

常见参数：

```python
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
```

---

### 6.6 AdamW

AdamW 是训练 Transformer 和大语言模型时非常常见的优化器。

它改进了 Adam 中 weight decay 的处理方式。

常用写法：

```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-4,
    weight_decay=0.01
)
```

AdamW 的优势：

* 比 Adam 更适合带权重衰减的训练
* 在 Transformer、BERT、GPT、ViT 中非常常见
* 泛化效果通常更好

---

## 7. Adam 和 AdamW 的区别

很多人容易混淆 Adam 和 AdamW。

核心区别是：

> AdamW 将 weight decay 从梯度更新中解耦出来。

普通 Adam 中，L2 正则会混入梯度计算；
AdamW 中，weight decay 是单独作用在参数上的。

简单理解：

| 优化器            | 特点                       |
| -------------- | ------------------------ |
| Adam           | 自适应学习率，收敛快               |
| AdamW          | Adam + 更合理的 weight decay |
| SGD            | 简单稳定，泛化可能较好              |
| SGD + Momentum | 比普通 SGD 更快更稳             |

现在训练 Transformer 通常优先考虑：

```text
AdamW
```

## Muon 优化器
【Muon：极简优化器，重塑 AI 训练范式 - Jia-Bin Huang】 https://www.bilibili.com/video/BV17ndFBNEnG/?share_source=copy_web&vd_source=c85963c68edeae09bd8fd784312c5ebc

Muon 全称是：

```text
MomentUm Orthogonalized by Newton-Schulz
```

可以理解为：

> Muon = SGD with Momentum + 矩阵更新方向的正交化处理

它主要用于神经网络中的 **二维权重矩阵**，尤其是 Transformer 里的 hidden layer 权重，比如 attention projection、MLP projection 等。原始 Muon 说明中也强调，它适合 hidden weights，而 embedding、classifier head、bias、gain 等参数通常仍然用 AdamW 处理。([GitHub][1])

---

### 1. Muon 想解决什么问题？

普通 SGD / AdamW 的更新方向可能会集中在少数几个主方向上。

也就是说，对于一个矩阵参数 (W)，它的梯度更新矩阵可能近似是“低秩”的：

```text
大部分更新量集中在少数方向
很多其他方向几乎没有被有效更新
```

Muon 的想法是：

> 不直接使用原始 momentum update，而是先把这个更新矩阵“正交化”，让不同方向的更新更加均衡。

Keller Jordan 的 Muon 介绍里提到，Transformer 中 SGD-momentum 和 Adam 对二维参数产生的更新通常条件数很高，接近低秩，Muon 通过正交化可能增强那些幅度较小但重要的方向。([Keller Jordan][2])

---

### 2. Muon 的核心流程

假设某一层的矩阵参数是：

\[+
W \in \mathbb{R}^{m \times n}
\]

当前梯度是：

\[
G_t = \nabla_W L
\]

Muon 的大致流程是：

#### 第一步：先做 Momentum

\[
M_t = \mu M_{t-1} + G_t
\]

其中：

* (M_t)：momentum buffer
* (\mu)：动量系数，常见值是 0.95
* (G_t)：当前梯度

这一步和 SGD with Momentum 很像。

---

#### 第二步：对 momentum update 做正交化

Muon 不直接用 (M_t) 更新参数，而是对 (M_t) 做近似正交化：

\[
O_t = \text{Ortho}(M_t)
\]

直观理解：

```text
原始更新矩阵 M_t
    ↓
Newton-Schulz 迭代
    ↓
近似正交化后的更新矩阵 O_t
```

如果用 SVD 来理解，假设：

\[
M_t = U \Sigma V^T
\]

普通梯度更新会保留奇异值 (\Sigma) 的大小差异。
Muon 的目标近似相当于把更新变成：

\[
O_t \approx UV^T
\]

也就是：

> 保留方向 (U,V)，但把不同方向上的尺度拉平。

原始 Muon 说明中也把它解释为：用 Newton-Schulz 迭代把 SGD-momentum 的更新矩阵替换为最近的半正交矩阵；等价理解是把 (USV^T) 替换成 (UV^T)。([Keller Jordan][2])

---

#### 第三步：更新参数

基本更新形式是：

\[
W_t = W_{t-1} - \eta O_t
\]

如果加入 weight decay，可以写成：

\[
W_t = W_{t-1} - \eta (O_t + \lambda W_{t-1})
\]

在大规模 LLM 训练的 Muon scaling 工作中，作者指出 weight decay 和 per-parameter update scale 对 Muon 扩展到大模型比较关键。

---

### 3. 为什么不用 SVD，而用 Newton-Schulz？

从数学上说，用 SVD 得到 (UV^T) 很直接。

但是 SVD 太慢，尤其是大模型里有大量矩阵参数。

所以 Muon 使用 **Newton-Schulz iteration** 来近似正交化矩阵。

大致形式可以理解为：

\[
X_{k+1} = aX_k + b(X_kX_k^T)X_k + c(X_kX_k^T)^2X_k
\]

其中 (a,b,c) 是系数。Muon 常见设置中使用类似：

```text
a = 3.4445
b = -4.7750
c = 2.0315
```

大规模 Muon 报告中也沿用了这个系数组合，并提到实际中常用 5 步 Newton-Schulz 迭代以兼顾效率。

---

### 4. Muon 和 AdamW 的区别

| 对比项               | AdamW                      | Muon                             |
| ----------------- | -------------------------- | -------------------------------- |
| 更新粒度              | 元素级 adaptive learning rate | 矩阵级正交化更新                         |
| 核心状态              | 一阶矩 + 二阶矩                  | momentum buffer                  |
| 主要思想              | 根据历史梯度大小调整每个参数学习率          | 对矩阵更新方向做正交化                      |
| 适用参数              | 几乎所有参数                     | 主要适合二维 hidden weight             |
| Transformer 中常见用法 | 全模型常用                      | hidden matrix 用 Muon，其他参数用 AdamW |
| 优点                | 稳定、通用、成熟                   | 可能更快收敛，尤其是矩阵权重                   |
| 风险                | 训练成本较高                     | 使用范围和超参数更需要注意                    |

一句话区别：

> AdamW 是“逐元素自适应”，Muon 是“矩阵整体几何修正”。

AdamW 关注的是每个参数自己的历史梯度尺度；
Muon 关注的是一个权重矩阵整体的更新方向是否过于集中。

---

### 5. Muon 在 Transformer 里优化哪些参数？

以 Transformer 为例，Muon 通常适合这些二维矩阵：

```text
Attention:
W_q, W_k, W_v, W_o

MLP / FFN:
W_up, W_gate, W_down

其他 hidden linear weights:
各种中间层 projection matrix
```

不太适合直接用于：

```text
embedding
lm_head / classifier head
bias
LayerNorm weight
RMSNorm weight
一维 scale 参数
```

原始 Muon repo 的用法也是把 hidden weights 设置为 `use_muon=True`，而 hidden gains/biases、head、embed 等参数设置为 `use_muon=False`，交给 AdamW。([GitHub][1])

---

### 6. 直观例子：为什么正交化有用？

假设一个矩阵更新方向的奇异值是：

```text
[100, 10, 1, 0.1, 0.01]
```

这说明更新几乎都集中在第一个方向。

普通优化器可能主要沿着最大方向更新，其他方向学得很慢。

Muon 做正交化后，相当于把不同方向的尺度拉平：

```text
[1, 1, 1, 1, 1]
```

当然这只是直观理解，实际实现是近似的。

这样做的好处是：

> 不让少数大方向完全支配更新，让更多方向都能参与学习。

这也是 Muon 经常被解释为一种 **spectral flattening** 或矩阵几何意义上的优化方法。

---

### 7. Muon 的优点

#### 7.1 收敛可能更快

Muon 在一些语言模型训练实验中表现出比 AdamW 更高的训练效率。比如 Moonshot AI 的大规模 Muon 报告称，在 scaling law 实验中，Muon 相比 AdamW 约有 (2\times) 计算效率，并在相近效果下需要约 52% 的训练 FLOPs。这个结论来自该技术报告的实验设置，实际效果仍取决于模型、数据和训练配置。

#### 7.2 更适合矩阵参数

Transformer 的核心参数大多是矩阵：

```text
线性投影矩阵
attention 矩阵
MLP 矩阵
```

Muon 正好针对这类二维参数设计。

#### 7.3 显存状态可能比 AdamW 少

AdamW 通常维护：

```text
参数本身
一阶矩 m
二阶矩 v
```

Muon 主要维护 momentum buffer，不像 AdamW 那样维护逐元素二阶矩，所以在某些实现和设置下可能有优化器状态优势。不过分布式训练下还要考虑矩阵正交化带来的通信与实现开销。

---

### 8. Muon 的缺点和注意事项

#### 8.1 不是所有参数都适合 Muon

Muon 主要针对二维矩阵参数。
对于 bias、LayerNorm、embedding、head 等参数，通常还是 AdamW 更稳。

#### 8.2 学习率不能直接照搬 AdamW

Muon 的学习率尺度和 AdamW 不一样。

例如原始 repo 示例中，hidden weights 的 Muon 学习率示例是：

```text
lr = 0.02
```

而 AdamW 部分示例是：

```text
lr = 3e-4
```

这说明 Muon 的 lr 量级可能和 AdamW 差很多，不能简单拿 AdamW 的学习率直接替换。([GitHub][1])

#### 8.3 大规模训练需要额外技巧

Muon 扩展到 LLM 时，weight decay、update scale、分布式实现都很重要。相关大规模报告指出，vanilla Muon 在更长训练中可能出现权重和 layer output RMS 增长过大的问题，因此加入 AdamW 风格 weight decay 有助于稳定训练。

---

### 9. 简化版伪代码

```python
# 对一个二维权重矩阵 W
# G: 当前梯度
# M: momentum buffer

M = momentum * M + G

# 可选：Nesterov-style momentum
update = momentum * M + G

# Newton-Schulz 近似正交化
O = orthogonalize_by_newton_schulz(update)

# 参数更新
W = W - lr * O
```

如果加 weight decay：

```python
W = W - lr * (O + weight_decay * W)
```

---

## 8. 学习率 Learning Rate

学习率决定每次参数更新的步子大小。

更新公式：

\[
\theta = \theta - \eta \nabla_\theta L
\]

其中 (\eta) 就是学习率。

学习率太大：

```text
loss 可能震荡、不收敛、甚至变成 NaN
```

学习率太小：

```text
loss 降得很慢，训练时间很长，可能卡在局部区域
```

一般经验：

| 场景             | 常见学习率                 |
| -------------- | --------------------- |
| 从零训练小模型        | (10^{-3})             |
| 微调 Transformer | (10^{-5}) 到 (10^{-4}) |
| AdamW 训练深度模型   | (10^{-4}) 到 (10^{-3}) |
| SGD 训练 CNN     | (10^{-2}) 到 (10^{-1}) |

这些只是经验值，实际要根据任务调整。

---

## 9. Learning Rate Scheduler

训练过程中，学习率通常不是固定不变的。

Scheduler 用来动态调整学习率。

常见策略：

### 9.1 Step Decay

每隔若干 epoch，学习率乘一个系数。

```text
lr = 0.1
训练 30 epoch 后变成 0.01
再训练 30 epoch 后变成 0.001
```

---

### 9.2 Cosine Decay

学习率按照余弦曲线逐渐下降。

常用于 Transformer、ViT、对比学习等任务。

特点：

```text
前期学习率较大，后期逐渐变小
```

---

### 9.3 Warmup

Warmup 指的是训练初期学习率从很小逐渐升高。

Transformer 训练中非常常见。

原因是：

> 模型刚开始参数很不稳定，如果学习率一上来太大，容易训练崩掉。

常见策略：

```text
前几千 step 线性增大学习率
之后再逐渐衰减
```

例如：

```text
warmup 阶段：0 → 1e-4
decay 阶段：1e-4 → 0
```

---

## 10. 反向传播 Backpropagation

训练时，模型会计算损失：

[
L
]

然后通过反向传播计算梯度：

[
\frac{\partial L}{\partial \theta}
]

这个梯度表示：

> 当前参数变化一点点，会让 loss 变化多少。

在 PyTorch 中一般是：

```python
loss.backward()
```

它会自动计算所有可训练参数的梯度。

---

## 11. 一个标准 PyTorch 训练流程

```python
for epoch in range(num_epochs):
    for x, y in dataloader:
        # 1. 前向传播
        logits = model(x)

        # 2. 计算损失
        loss = criterion(logits, y)

        # 3. 清空旧梯度
        optimizer.zero_grad()

        # 4. 反向传播
        loss.backward()

        # 5. 更新参数
        optimizer.step()
```

注意这里的顺序非常重要：

```text
forward → loss → zero_grad → backward → step
```

或者更常见写法：

```python
optimizer.zero_grad()
logits = model(x)
loss = criterion(logits, y)
loss.backward()
optimizer.step()
```

---

## 12. 为什么要 optimizer.zero_grad()？

PyTorch 中梯度默认是累加的。

如果不清空梯度：

```python
loss.backward()
loss.backward()
loss.backward()
```

梯度会不断叠加，而不是覆盖。

所以每次更新前通常要写：

```python
optimizer.zero_grad()
```

否则模型更新会出问题。

---
## 13. Batch、Step、Epoch 的关系

| 概念               | 含义           |
| ---------------- | ------------ |
| Batch            | 一次送入模型的一小批数据 |
| Step / Iteration | 参数更新一次       |
| Epoch            | 完整遍历一遍训练集    |

例如：

```text
训练集大小 = 10,000
batch size = 100
```

那么：

```text
每个 epoch 有 100 个 step
```

因为：

[
10000 / 100 = 100
]

如果训练 10 个 epoch：

```text
总共更新 1000 次参数
```

---

## 14. 正则化 Regularization

正则化用于缓解过拟合。

过拟合指的是：

```text
训练集表现很好
验证集/测试集表现很差
```

常见正则化方式包括：

### 14.1 Weight Decay

Weight decay 会限制参数过大。

直观理解：

> 不希望模型用特别大的权重去强行记忆训练集。

在 AdamW 中常用：

```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-4,
    weight_decay=0.01
)
```

---

### 14.2 Dropout

Dropout 会在训练时随机丢弃一部分神经元。

例如：

```python
nn.Dropout(p=0.1)
```

含义是：

```text
训练时随机让 10% 的神经元输出为 0
```

作用：

* 减少神经元之间的过度依赖
* 提升模型泛化能力

---

### 14.3 Label Smoothing

Label smoothing 常用于分类任务。

普通 one-hot 标签：

```text
正确类别：1.0
其他类别：0.0
```

Label smoothing 后：

```text
正确类别：0.9
其他类别：0.1 / (类别数 - 1)
```

作用：

* 防止模型过度自信
* 提高泛化能力
* 常用于 Transformer 分类、机器翻译等任务

---

## 15. 梯度消失与梯度爆炸

### 15.1 梯度消失

梯度太小，导致前面层几乎学不到东西。

表现：

```text
loss 几乎不下降
参数更新非常慢
深层网络训练困难
```

常见缓解方式：

* ReLU 类激活函数
* 残差连接
* LayerNorm / BatchNorm
* 合理初始化

---
### 15.2 梯度爆炸

梯度太大，导致参数更新剧烈。

表现：

```text
loss 突然变成 NaN
训练不稳定
参数数值异常大
```

常见缓解方式：

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

这叫 **gradient clipping**，梯度裁剪。

Transformer 和 RNN 训练中经常使用。

---

## 16. 梯度裁剪 Gradient Clipping

梯度裁剪的作用是：

> 当梯度范数超过阈值时，把它缩放回合理范围。

例如：

```python
loss.backward()

torch.nn.utils.clip_grad_norm_(
    model.parameters(),
    max_norm=1.0
)

optimizer.step()
```

它不是改变优化目标，而是防止某一步更新过大，导致训练崩掉。

---

## 17. 分类任务中的 logits、softmax 和 loss

这是非常重要的知识点。

模型最后输出通常叫 logits：

```text
logits = [2.1, 0.3, -1.2]
```

logits 还不是概率。

经过 softmax 后才是概率：

```text
probs = [0.82, 0.14, 0.04]
```

但是在 PyTorch 中：

```python
criterion = nn.CrossEntropyLoss()
loss = criterion(logits, labels)
```

这里传入的是 logits，不是 softmax 后的概率。

不要写成：

```python
probs = softmax(logits)
loss = CrossEntropyLoss(probs, labels)
```

这样通常是不推荐的，因为 `CrossEntropyLoss` 内部已经做了 log-softmax 相关计算。

## 18. 回归、二分类、多分类、多标签如何选 loss？

| 任务类型  | 输出形式           | 常用损失函数                          |
| ----- | -------------- | ------------------------------- |
| 回归    | 连续值            | MSELoss / L1Loss / SmoothL1Loss |
| 二分类   | 一个 logit       | BCEWithLogitsLoss               |
| 多分类   | 多个 logits，单一类别 | CrossEntropyLoss                |
| 多标签分类 | 多个独立标签         | BCEWithLogitsLoss               |
| 语言模型  | 每个位置预测词表 token | CrossEntropyLoss                |
| 对比学习  | 表示相似度          | InfoNCE / Contrastive Loss      |

## 19. Smooth L1 Loss / Huber Loss

Smooth L1 Loss 结合了 MSE 和 MAE 的优点。

误差小时像 MSE：

```text
更平滑
```

误差大时像 MAE：

```text
对异常值不那么敏感
```

常用于目标检测中的边界框回归。

## 20. 训练过程常见现象

### 20.1 训练 loss 下降，验证 loss 也下降

说明模型在正常学习。

---

### 20.2 训练 loss 下降，验证 loss 上升

说明可能过拟合。

解决方法：

* 增加数据
* 使用 dropout
* 使用 weight decay
* 早停 early stopping
* 数据增强
* 降低模型复杂度

---

### 20.3 训练 loss 不下降

可能原因：

* 学习率太小
* 数据标签有问题
* loss 选错了
* 模型结构有 bug
* 梯度没有正常反传
* 参数没有加入 optimizer
* 输入没有归一化

---

### 20.4 loss 变成 NaN

可能原因：

* 学习率太大
* 梯度爆炸
* log(0)
* 除以 0
* 输入数据存在 NaN
* mixed precision 溢出

解决方式：

* 降低学习率
* 加 gradient clipping
* 检查数据
* 检查 loss 计算
* 使用更稳定的 loss 实现
* 检查 AMP 设置

## 22. 一个完整训练例子

以多分类任务为例：

```python
import torch
import torch.nn as nn
import torch.optim as optim

model = MyModel()

criterion = nn.CrossEntropyLoss()

optimizer = optim.AdamW(
    model.parameters(),
    lr=1e-4,
    weight_decay=0.01
)

for epoch in range(num_epochs):
    model.train()

    for x, y in train_loader:
        optimizer.zero_grad()

        logits = model(x)

        loss = criterion(logits, y)

        loss.backward()

        torch.nn.utils.clip_grad_norm_(
            model.parameters(),
            max_norm=1.0
        )

        optimizer.step()

    print(f"epoch={epoch}, loss={loss.item()}")
```

核心流程：

```text
清空梯度
前向传播
计算损失
反向传播
梯度裁剪
参数更新
```

## 23. 面试常问问题

### Q1：损失函数的作用是什么？

损失函数衡量模型预测和真实标签之间的差距，并为反向传播提供优化目标。

---

### Q2：优化器的作用是什么？

优化器根据损失函数计算出的梯度更新模型参数，使 loss 尽可能下降。

---

### Q3：CrossEntropyLoss 为什么不用先 softmax？

因为 PyTorch 的 `CrossEntropyLoss` 内部已经包含了 `LogSoftmax + NLLLoss`，直接输入 logits 更稳定。

---

### Q4：学习率太大会怎样？

学习率太大会导致参数更新步子过大，loss 震荡、不收敛，甚至出现 NaN。

---

### Q5：学习率太小会怎样？

学习率太小会导致训练速度很慢，loss 下降缓慢，可能在有限训练时间内学不到有效模型。

---

### Q6：Adam 和 SGD 有什么区别？

SGD 按照当前梯度更新参数，简单但可能收敛慢。
Adam 使用一阶矩和二阶矩估计，为不同参数自适应调整学习率，通常收敛更快。

---

### Q7：AdamW 为什么常用于 Transformer？

因为 AdamW 在 Adam 的基础上更合理地处理 weight decay，训练 Transformer 时通常更稳定，泛化也更好。

---

### Q8：为什么要梯度裁剪？

梯度裁剪用于防止梯度爆炸，避免某一步参数更新过大导致训练不稳定或 loss 变成 NaN。
